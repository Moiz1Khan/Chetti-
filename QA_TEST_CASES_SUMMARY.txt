PART 1 — WHAT WE DID
--------------------

OBJECTIVE
  Full QA and end-to-end (E2E) testing for Data Smart Chat: authentication,
  dashboard, API integrations, UI/UX responsiveness, error handling, and
  user flows from start to finish.

TEST STACK
  - Runner: Playwright (Node.js)
  - Config: playwright.config.ts — base URL http://localhost:8080, 1 worker,
    90s timeout per test, trace/video off, screenshot only on failure
  - Helpers:
    * Auth: loginAs(page, "user"|"admin"), logout(page). Uses QA_USER_EMAIL,
      QA_USER_PASSWORD, QA_ADMIN_EMAIL, QA_ADMIN_PASSWORD when set; else signup.
    * Seed: seedChatbotWithTextKnowledge(page, "user") — creates knowledge +
      chatbot via UI with QA_Test_ prefix.
    * Stream: waitForAssistantMarkdownNonEmpty(page) — waits for assistant text.

SECURITY (API KEYS)
  - No trace/video; screenshot only on failure globally.
  - API spec has screenshot/trace/video off so key dialogs are never captured.
  - No raw API keys or credentials in any report or this file.

FIXES APPLIED
  1. beforeAll + page: Playwright does not allow page in beforeAll. We moved
     seeding into the first test of each serial suite (dashboard, knowledge-base,
     public-chatbot, api).
  2. Landing "has navigation links": Selector updated to a[href="/login"] for
     stability (link can be wrapped in button).
  3. Test timeout: Increased to 90s in config so login (60s wait) can complete.

HOW TO RUN
  1. Start app: npm run dev (http://localhost:8080)
  2. Optional env: QA_USER_EMAIL, QA_USER_PASSWORD, QA_ADMIN_EMAIL, QA_ADMIN_PASSWORD
  3. First time: npx playwright install chromium
  4. Run: npm run e2e  or  npx playwright test
  5. Report: npx playwright show-report
